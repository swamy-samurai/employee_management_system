# 📊 Employee Management System – MySQL Data Analytics Project  

## 📌 Project Overview  
This project focuses on building a *relational Employee Management System* using MySQL and performing *data analysis* to extract meaningful business insights from HR, payroll, and leave data.  

The system is designed with proper *database normalization* and includes multiple analytical queries to understand employee trends, salary distribution, and organizational patterns.  

---

## 🎯 Objectives  
- Design a structured relational database  
- Implement normalization techniques  
- Analyze employee, payroll, and leave data  
- Generate business insights using SQL queries  

---

## 🛠 Tech Stack  
- *SQL (MySQL)*  
- Database Design & Normalization  
- MySQL Workbench  

---

## 🗂 Database Structure  
The project consists of *6 normalized tables*:  
- Employee  
- JobDepartment  
- SalaryBonus  
- Qualification  
- Leaves  
- Payroll  

✔ Applied Constraints:  
- Primary Keys  
- Foreign Keys (with CASCADE / SET NULL)  
- UNIQUE constraint on email  
- Standard DATE format (YYYY-MM-DD)  

---

## 📊 Key Analysis Performed  
- Employee distribution across departments  
- Average salary per department  
- Top 5 highest-paid employees  
- Total payroll expenditure  
- Job role and department insights  
- Qualification and skill analysis  
- Leave and absence patterns  
- Payroll and bonus analysis  

---

## 💡 Key Insights  
- Total *60 employees* across *10+ departments*  
- *Finance department* has highest employee count and salary allocation  
- Highest paying role: *Finance Director*  
- All employees have at least one qualification  
- Leave data shows consistent employee participation  
- Finance department leads in both salary and bonus distribution
